# Stage 2: Dashboard-first onboarding

**Goal:** Make the first-run experience centered on the organization dashboard.

**Focus:** Landing view, app creation, API keys, connection setup, and alert endpoint management.

## Features
- Feature 3: Organization overview and onboarding guidance (`../features/03-org-overview-and-onboarding.md`)
- Feature 4: App creation, API key generation, and connection setup (`../features/04-app-creation-api-key-connection.md`)
- Feature 5: Alert endpoint management and alert configuration (`../features/05-alert-endpoint-management.md`)

## Exit criteria
- New users land on the org overview, not a dead-end checklist.
- The dashboard path naturally leads from org -> app -> key -> package.
- Alert setup is visible in the dashboard flow, not hidden later.
