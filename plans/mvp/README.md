# Crontinel MVP Planning Workspace

Status: intake started

## Rules for this workspace
- Ask **one question at a time**.
- After each answer, update the plan before asking the next question.
- Keep a master plan, stage plans, feature plans, and task breakdowns.
- Track progress in a separate file so the status stays visible.
- Do not start implementation until the plan is written.

## Files
- `docs/plans/mvp/master-plan.md` - top-level MVP plan
- `docs/plans/mvp/progress.md` - progress tracker
- `docs/plans/mvp/questions.md` - running intake log
- `docs/plans/mvp/implementation-tracker.md` - live task/worker tracker and impediment log
- `docs/plans/mvp/runtime/` - live worker state and heartbeat files
- `docs/plans/mvp/stages/` - stage plans, one file per stage
- `docs/plans/mvp/features/` - feature plans, one file per feature
- `docs/plans/mvp/tasks/` - task breakdowns, one file per feature or task group
